# IRkun リデザイン要望書（2026-02-22）

## A. そもそもの目的

### 何を共有するのか（情報）
1. **時系列を追う**: これまでの障害福祉の歴史・制度変遷・企業動向を時系列で把握
2. **最新情報の収集・更新**: メンバーから拾い上げた最新情報が反映され、普段の意思決定に活用

### 誰が見るのか
1. **障害福祉支援部内で**:
   - **新人**: 業界のキャッチアップが進むもの（入門・学習用途）
   - **企画者**: 事業の意思決定や探索領域の設定（戦略立案用途）
     - → 情報の鮮度があいまいなものはそれがわかるとよい

### いつ見るのか（トリガー）
- 決算説明資料が出たタイミング
- 新しい企画を担当することになった時の簡易デスクトップリサーチ
- 新入社員が早期でキャッチアップするタイミング
- その他イベントごと

## B. 情報構造の要望（マクロ→ミクロ）

### レイヤー1: マクロ（最上位）
**日本における障害福祉の沿革**
- 障害者雇用促進法、障害者総合支援法 etc.
- 海外の障害福祉の沿革との比較
- 介護の法律と障害の法律の類似性

### レイヤー2: 3つのプレイヤーの分布
**利用者・企業・事業者**
- それぞれのプレイヤーがいつ・なぜ増えたのかを深堀り
- その一つのアウトプットが「事業所分析」

### レイヤー3: 事業所分析（各サービス種別ごと）
- 参入している法人プレイヤー、成長率、報酬改定の時系列
- 一日の流れ、登場人物
- **ステークホルダーとの関わり（図式化！やり取りが鮮明に見える形で）**
- 普段の会話のシーン

#### ★ 新規要望: 事業所分析の深堀り
1. **利用者の一連の流れ**:
   - 利用者が来て → 記録を残し → 請求するまでの流れ
   - 新規利用者: 手続き → 学習内容 → モニタリング → 卒業まで
2. **開業の一連の流れ**:
   - 代表が開業を決意 → 必要書類・審査手続き → 申請受理
   - 開業資金: いくら必要か、融資先、融資額
   - 自治体の総量規制、独自フォーマット申請
   - 利用者募集のアクション、土地決定のアクション
   - 各サービス種・地域・開業背景ごとの違い
3. **加算の深堀り**:
   - 現状はまとめただけ → もっと深く
   - 処遇改善加算の取得は段階的で複雑 → その複雑さを表現

### レイヤー4: 企業分析
**主要プレイヤーの動向（SaaS / 人材 / メディア / その他事業）**

#### 上場企業（詳細）
- 市場規模、その領域
- 一覧: 時価総額 / 売上 / 事業セグメント / 営業利益率
- 会社全体の BS / PL / CF の時系列
- 事業セグメント別: 管理会計的に売上構成要素・コスト構成要素
  - → **エクセルを動かすことで収支がどう変わるのか** までシミュレーション可能に

#### 非上場企業
- Webリサーチベースの情報（Tavily経由）

## C. サイト構造の改善要望

### 1. 情報過多対策

- マクロ→ミクロの導線を明確に

### 2. ステークホルダーマップの図式化
- 表ではなく図式化でやり取りが鮮明に

### 3. 事業計画シミュレーション
- エクセル的にパラメータを動かして収支変動を見れる
